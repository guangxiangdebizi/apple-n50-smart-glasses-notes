# 蓝牙 · UWB 通信链路

> **免责声明**：蓝牙/UWB 能力基于苹果现有技术栈推演，**非 N50 官方通信规格**。

---

## 模块概述

N50 与 iPhone 之间的链路是三层算力的「数据动脉」。蓝牙 LE 承担常规控制与音频；传闻 **UWB（U1/U2 芯片）** 提供更低延迟、更高吞吐的空间数据通道，支撑 Visual AI 实时性与手势/空间交互。

---

## N50 传闻规格

| 协议 | 用途（推演） | 置信度 |
|------|-------------|--------|
| 蓝牙 LE | 音频（A2DP/LE Audio）、控制信令、状态同步 | 高 |
| UWB | 低延迟视觉特征传输、空间定位、精确配对 | 中 |
| Wi-Fi | 眼镜端**不主打**独立 Wi-Fi | 高 |
| 带宽需求 | 视觉特征流 ~1–5 Mbps（推演） | 低 |
| 延迟目标 | 传输 <50ms；端到端 <150ms | 低 |
| 芯片 | 眼镜集成 UWB；iPhone U1/U2 | 中 |

**对比 Meta Ray-Ban**：Gen 2 使用 Wi-Fi + BT，镜端 AR1 可独立处理更多数据，对手机链路依赖较低。

## 供应商 / 生态格局

| 组件 | 厂商 | 说明 |
|------|------|------|
| BT 芯片 | 苹果/Broadcom | iPhone 标配；眼镜集成于 N401 |
| UWB | 苹果 U1/U2 | AirTag/Handoff/精确查找已验证 |

| 标准 | IEEE 802.15.4z (UWB) | FiRa 联盟成员 |
| 竞品 | 高通 Wi-Fi/BT 组合 | AR1 平台标准方案 |

## 供应链或系统逻辑

```
┌──────── N50 眼镜 ────────┐     ┌────── iPhone ──────┐
│ N401 + BT/UWB 射频       │     │ A 系列 + U1/U2     │
│ 摄像头→ISP→特征编码      │────→│ NE 推理            │
│ OWS ←── 音频下行 ────────│←────│ Siri TTS           │
└──────────────────────────┘     └────────────────────┘
         BT LE：控制 + 音频（常规路径）
         UWB：  高优先级视觉数据/空间同步（传闻）
```

**链路设计推演**：
1. **配对**：UWB 精确测距 + BT 标准配对（类似 AirPods 体验）
2. **视觉 AI 模式**：UWB 传低延迟特征包；BT 传元数据/心跳
3. **省电模式**：仅 BT 维持连接，UWB 按需唤醒
4. **断连恢复**：iPhone 缓存最近上下文，快速重连

---

## 优势分析

- **UWB 低延迟**：理论 ms 级 TOF，适合空间交互与突发数据
- **UWB 高吞吐**：短距宽带传输，缓解 BT 带宽瓶颈
- **苹果独有**：Android XR 眼镜难复刻 UWB + NE 深度整合
- **对比 Wi-Fi**：UWB 功耗更低，无需眼镜独立 Wi-Fi 天线
