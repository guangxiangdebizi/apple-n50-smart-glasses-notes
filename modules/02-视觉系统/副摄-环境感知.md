# 副摄 · 环境感知（CV / 手势 / 上下文）

> **非官方说明**：双摄像布局来自 Gurman 等公开传闻；副摄分辨率、具体传感器型号**无官方披露**。

---

## 1. 模块概述

副摄是 N50 **低功耗感知通道**：以**低分辨率广角**持续采集环境上下文，支撑计算机视觉（CV）、手势识别、头部/佩戴状态、端侧事件预筛选，减轻主摄频繁唤醒带来的功耗与隐私 LED 打扰。

分工逻辑类似手机「主摄 + 超广角/深度」，但在眼镜上更强调 **always-on 低带宽** 与 **N401 端侧 ISP 预筛选**。

---

## 2. N50 传闻规格

| 参数 | 传闻值 | 置信度 |
|------|--------|--------|
| 分辨率 | **低分辨率**（业界推测 VGA–2MP 级） | 中 |
| 视场 | **广角**，覆盖佩戴者前方扇区 | 中 |
| 功能 | CV、手势、环境上下文、可能配合头部动作 | 中 |
| 供应商 | **索尼** 等（与主摄同厂概率高） | 中 |
| 输出 | 特征/元数据优先上传 iPhone，非全分辨率流 | 低（推演） |

## 4. 供应链逻辑

副摄 BOM 单价低于主摄，但 **算法与校准成本** 不低：

```
低像素 CIS（索尼/OV）→ 更简单塑胶广角镜头 → 可能与主摄共模组厂封装
    → N401 ISP / CV 加速器（端侧）
    → 仅上传事件/骨架/深度提示 → iPhone Neural Engine
```

苹果若要求 **副摄常开**，功耗须 **<< 主摄录像**；产业链倾向 **全局快门或高帧率小图** 用于手势，而非 4K 流。

---

## 5. 厂商优势

**索尼（若双索尼）**

- 双摄色彩/时间戳同步校准经验来自 iPhone 多摄。
- 统一 SDK 降低 N401 驱动复杂度。

**OmniVision 等二线（成本方案）**

- Meta 已验证 OV5675；成本与供货灵活。
- 苹果高端线可能仍优先索尼以保一致性与良率。

---

## 6. 前沿未解问题

1. **常开 CV 与隐私 LED**：副摄是否触发指示灯？法规（EU/州法）仍演化中。
2. **手势 vs 触控 vs 语音**：副摄手势是否重复 Apple Watch 双指手势？
3. **算力边界**：复杂 CV 在 N401 还是 iPhone？蓝牙带宽是否够实时骨架？
4. **双副摄 vs 单副摄**：Meta 用 2×5MP；苹果传闻仅「副摄」单数，配置未定。
5. **低光可穿戴**：夜间室内 gesture 识别是否需 IR 补光（增加镜框厚度）。
